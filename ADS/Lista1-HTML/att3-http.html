Protocolo HTTP

O que é o HTTP?

O HTTP (HyperText Transfer Protocol) é o protocolo que permite a comunicação entre navegadores e servidores na Web. Sempre que um usuário acessa um site, o navegador envia uma solicitação ao servidor, que responde enviando os recursos necessários, como páginas HTML, imagens, vídeos e outros arquivos.

Papel do HTTP na comunicação entre navegador e servidor

O HTTP define as regras para a troca de informações entre o cliente e o servidor. Quando um endereço é acessado, o navegador envia uma requisição ao servidor. Em seguida, o servidor processa essa solicitação e retorna uma resposta contendo o conteúdo solicitado ou uma mensagem indicando algum erro.

Cliente, Servidor, Requisição e Resposta

Cliente

É o dispositivo ou programa responsável por solicitar informações ao servidor. Geralmente, trata-se de um navegador, como Google Chrome, Mozilla Firefox ou Microsoft Edge.

Servidor

É o computador que hospeda o site e fornece os recursos solicitados pelos usuários.

Requisição (Request)

É a solicitação enviada pelo cliente ao servidor para acessar um determinado recurso, como uma página da Web ou uma imagem.

Resposta (Response)

É a mensagem enviada pelo servidor em resposta à requisição. Ela pode conter o recurso solicitado ou um código informando o resultado da solicitação.

Métodos HTTP

GET

O método GET é utilizado para solicitar informações ao servidor. Ele serve para acessar páginas, consultar dados e obter recursos, sem modificar as informações existentes.

POST

O método POST é usado para enviar informações ao servidor. É muito utilizado em formulários de cadastro, login, envio de mensagens e upload de arquivos.

Exemplo de uma Requisição HTTP

        GET /index.html HTTP/1.1
        Host: www.exemplo.com
        User-Agent: Mozilla/5.0
        Accept: text/html

Exemplo de uma Resposta HTTP

        HTTP/1.1 200 OK
        Content-Type: text/html
        
            
        Página carregada com sucesso!

Códigos de Status HTTP

200 - OK

Indica que a requisição foi processada com sucesso e o recurso solicitado foi entregue ao cliente.

301 - Moved Permanently

Informa que o recurso foi movido permanentemente para outro endereço. A partir desse momento, o navegador deve utilizar o novo endereço.

400 - Bad Request

Significa que a requisição foi enviada de forma incorreta e não pôde ser compreendida pelo servidor.

403 - Forbidden

O servidor entendeu a solicitação, mas o cliente não possui permissão para acessar o recurso solicitado.

404 - Not Found

Indica que o recurso solicitado não foi encontrado no servidor.

500 - Internal Server Error

Representa um erro interno do servidor, impedindo que a requisição seja concluída com sucesso.
